# SEO Skills Overview

## Skills

| スキル名 | 役割 | 使用目的 |
|---|---|---|
| `seo-column-pipeline` | 司令塔 | 1本のコラム記事を、企画からキーワード選定、SERP分析、brief作成、構成、本文、CTA、QAまで一気通貫で進める |
| `keyword-strategy` | 企画・優先度設計 | 狙うべきキーワード群、検索意図、事業接続、記事化優先度を決める |
| `serp-analysis` | 検索結果分析 | 上位記事の傾向、検索意図、競合パターン、不足論点、差別化余地を整理する |
| `content-outline` | 記事構成設計 | brief と SERP 分析をもとに、見出し構成、論点順序、FAQ、内部リンク方針を設計する |
| `seo-writer` | 本文執筆 | SEO・AIO・GEO を踏まえて、日本語のコラム本文、タイトル案、description 案、FAQ 案を書く |
| `conversion-cta` | CV導線設計 | 記事テーマと読者温度に合う CTA、内部導線、記事末尾の訴求文を設計する |
| `quality-audit` | 公開前監査 | 記事を SEO、正確性、AIO/GEO、ブランド整合性、CV 導線の観点でレビューし、公開可否を判定する |

## Supporting Files

| ファイル / ディレクトリ | 役割 | 使用目的 |
|---|---|---|
| `templates/article-brief.md` | brief 雛形 | 記事テーマ、読者、検索意図、勝ち筋、CTA 方針を整理する |
| `templates/outline-template.md` | 構成雛形 | タイトル、導入、H2/H3、FAQ、CTA までの骨子を揃える |
| `templates/qa-checklist.md` | QA 雛形 | 公開前チェックを抜け漏れなく行う |
| `briefs/` | 記事ごとの企画・指示書置き場 | 各記事の brief を保存する |
| `references/` | 調査メモ・根拠置き場 | 一次情報、競合観察、社内情報の整理に使う |
| `skills/` | skill 正本 | リポジトリ管理する skill 定義を置く |
| `~/.codex/skills` へのリンク | 実行用 | slash 形式で各 skill を呼び出せるようにする |
