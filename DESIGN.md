# DESIGN.md

## 目的

- このファイルは、サイト全体のデザイン実装ルールを定義する。
- 特にホームの 1st / 2nd / 3rd view は、スクリーンショット準拠の再現ルールをここで固定する。
- UI 改修時は、個別判断より先にこのファイルを参照する。

## 共通ルール

- セクションごとに高さの意図を持たせる。
- 1 view = 1 画面として扱う場合は、`svh` 基準で設計する。
- スクリーンショット準拠の実装では、優先順位を次の順に固定する。
  1. 余白と高さ
  2. 見出しの改行位置
  3. 配色
  4. 既存コンポーネント再利用
- 見出しの改行位置を固定したい場合は、文字列の自然折り返しに頼らず、行ごとに要素を分けて実装する。
- CTA は主従を明確に分ける。
  - Primary: 赤背景 + 白文字
  - Secondary: 黒背景 + 白文字
  - Tertiary: 文字リンクまたは白背景を使わない軽量導線
- CTA の角丸は原則つけない。矩形ベースで設計する。
- スクショ準拠のホームビューでは、英字ラベルは太く短く、日本語ラベルは補助情報として扱う。

## 非トップページ共通余白

- ホーム以外のページでは、セクション外枠と中身の余白をトークンで管理する。
- セクション内の上下余白は `--fx-page-section-padding-y` を基準にする。
- 連続する主要セクション同士の上方向の間隔は `--fx-page-section-gap-y` を基準にする。
- セクション見出しから最初の主要要素までの間隔は `--fx-page-content-gap-y` を基準にする。
- 非トップページの主要コンテンツ幅は `--fx-page-shell-max` を基準にし、個別理由がない限り独自幅を置かない。
- ABOUT / SERVICE のような専用ページも、ヒーロー下の主要セクション、一覧、導入フロー、採用情報などは上記トークンを使う。
- ホームのスクショ準拠ビューはこの共通余白の対象外とし、ホームビュー規則を優先する。

## コラム記事テンプレート

- コラム詳細は本文の読みやすさを優先し、記事本文と資料請求フォームの 2 カラムを基本にする。
- 本文カラムは長文を追いやすい幅に抑え、右カラムの資料請求フォームはデスクトップで追従表示する。
- 記事タイトル直下に公開日、タグ、読了目安を置き、リード文はサムネイル下、目次上に配置する。
- 目次は白背景と赤枠で、本文への導線として過度な装飾を避ける。ホバーとフォーカス時は赤文字で反応を示す。
- 記事本文の h2 は赤文字と赤アンダーライン、h3 は赤い左バーと薄い赤背景で階層差を出す。
- 本文中の強調は赤太字を基本とし、補足メモ、要点リスト、表はトーンを揃えて過度な罫線や影を避ける。

## ホームビュー規則

### 1st View

- 高さは 1 画面以内に収める。
- ナビゲーションは背景透過を基本とし、右端 CTA だけ色面を持たせる。
- ヒーロー見出しは指定改行を固定し、本文と CTA を左下にまとめる。
- メタボール表現は既存背景実装を流用してよい。

### 2nd View

- 高さは `100svh` を基準とする。
- 左から右へ抜けるグラデーション背景を持たせる。
- `ABOUT` の英字見出し、和文補助ラベル、大きい日本語見出し、本文、Primary CTA の順で並べる。
- 本文量が増えても 1 画面高を大きく超えないように、上下余白を先に詰める。

### 3rd View

- 高さは 2nd View の約 150% を基準とする。
- 白基調の背景で、グラデーションは使わない。
- 上部に `SERVICE` 英字見出し、和文補助ラベル、主見出し、ダミー本文、Primary CTA を置く。
- 下部は左側に事業説明ブロック、右側と下段に固定カード群を置く。
- カード群の文言はスクショ忠実で固定する。
- ホームの 3rd view は CMS 的な可変性を持たせず、ホーム専用コンポーネントで実装する。

## 実装規則

- ホーム専用のスクショ準拠レイアウトは、汎用コンポーネントへ条件分岐を増やしすぎない。
- 汎用性よりも再現性を優先すべき場合は、ホーム専用コンポーネントを新設する。
- 既存ページが依存するコンポーネントは、ホーム専用レイアウトの都合で破壊しない。
- デザイン意図が文書化されていない状態で、新しい view を追加しない。
